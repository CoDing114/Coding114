자주 쓰는 명령어

    수정중 WHERE문을 빼먹으면 재앙이 올수잇다!!!
  • Create; 생성
  • Drop; 삭제
  • SHOW ***; 데이터 베이스 조회
  • USE ***; 데이터 베이스 선택
  • DESC ***; 구조확인
  • SELECT * FROM topic;추가한 데이터 확인
  • ORDER BY id DESC; 높은 값 부터 정렬
  • LIMIT 2; 보이기 수량 설정가능
  • RENAME TABLE topic TO topic_backup; 이름 변경
  • UPDATE topic SET description='Oracle is...',title='Oracle' WHERE id=2; 수정
  • DELETE FROM topic WHERE id=5; 데이터 삭제

자주 쓰는 용어

  • Table:표
  • Row,record:행
  • column:열
 
실습내용

  경로선택    1:CD C:\Bitnami\wampstack-8.0.8-0\mariadb\bin   
  MYSQL실행   2:C:\Bitnami\wampstack-8.0.8-0\mariadb\bin  >mysql -uroot -p(비밀번호입력가능)
  DB생성      3:MariaDB [opentutorials]> create table topic( --입력후 ';'없이 Enter를 누르면 줄바꿈이된다
                -> id INT(11) NOT null AUto_increment, --INT(11)[인티저(보이기수량)],NOT null[공백 비허용],AUto_increment[자동으로 1씩 추가 중복을 회피]
  DB선택      4:USE opentutorials; 
  TABLE생성   5:
CREATE TABLE topic(
id INT(11)NOT NULL AUTO_INCREMENT,
title VARCHAR(45)NOT NULL,
description TEXT,
created DATETIME NOT NULL,
PRIMARY KEY(id))
ENGINE=InnoDB;
  TABLE확인   6:SHOW TABLES;
  TB구조 확인 7:DESC topic;
  데이터 추가 8:INSERT INTO topic_backup (title,description,created,author,profile) VALUES('DOGE','DOGE is....',NOW(),'Elon','Musk');
  데이터 확인 9:SELECT * FROM topic;
  데이터 변경10:UPDATE topic SET description='Oracle is...',title='Oracle' WHERE id=2;
